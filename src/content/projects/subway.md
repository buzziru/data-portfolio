## Problem

지하철 역별·시간대별 이용인원을 예측하는 문제. 예측 모델보다 데이터 수집과 통합이 핵심 과제였습니다. 흩어진 공공데이터를 의미 있는 분석 단위로 통합하는 전 과정을 팀리드로 주도했습니다.

## Data

4개 영역 공공데이터 직접 수집:
- **지하철**: 서울 열린데이터 · 역별·시간대별 이용인원
- **기후**: 공공데이터포털 · 기온·강수량·풍속
- **부동산**: 역 반경 거주 인구 및 상업지구 밀도
- **버스**: 버스 노선·정류장 데이터 (환승 패턴 피처)

총 350만 행 · 50개 변수 구축.

## Approach

1. **수집**: 4개 영역 API 및 파일 다운로드 자동화
2. **통합**: 역·시간대(날짜+시간) 복합 키로 테이블 조인
3. **전처리**: 결측 처리, 파생 피처(요일/공휴일/계절) 생성
4. **모델링**: 회귀 5종(Linear, Ridge, Random Forest, XGBoost, LightGBM) 비교 + 교차검증

## Model / Analysis

LightGBM이 비선형 시간대 패턴과 날씨 상호작용을 가장 잘 포착했습니다. 시간대별 피크(출퇴근) 패턴이 다른 요일·날씨 효과를 상호작용 피처로 명시했습니다.

## Results

- 350만 행 · 50개 변수 분석 데이터셋 직접 구축
- LightGBM이 회귀 5종 중 교차검증 기준 최우수 성능
- 수집·전처리·모델링·발표를 팀리드로 전 주도

## What I Learned

- 데이터 수집 자동화(API 호출·파일 파싱)가 분석 시간을 결정한다.
- 복합 키 조인 설계가 잘못되면 데이터 누수·편향이 발생 — 조인 키 검증이 필수.
- 350만 행 규모에서는 pandas만으로 한계 — MariaDB 중간 저장 전략이 효과적이었다.

## Next Improvements

딥러닝 시계열(LSTM·Transformer) 적용 · 실시간 예측 API 배포 · 노선 확장.
